# Prompt 1 Answer: Repository Purpose and Experiment Summary

The main purpose of this repository and experiment is to use `addon-tellsticklive-roosfork` — a Home Assistant add-on and custom integration for TellStick Duo hardware — as the first test project for `ha-dev-platform`, a generic MCP (Model Context Protocol) orchestrator designed to automate the full code→release→deploy→test→fix development loop against a live Home Assistant instance. The user's ultimate goal is to establish a reusable development platform that allows Copilot agents to autonomously create test releases, deploy add-ons via HACS, run integration tests, retrieve HA logs, and reset the environment, all driven through a Cloudflare-tunnelled API. The key infrastructure consists of a Home Assistant OS (HAOS) VM running on TrueNAS SCALE (required for Supervisor and add-on support), the `ha-dev-platform` Docker container pointed at that VM via a `.env` file, and a Cloudflare tunnel exposing the MCP endpoint — all while keeping the two repositories separate so that `ha-dev-platform` remains a generic, reusable service that any current or future project repo can consume by simply adding a small set of template config files. The expected result is a fully automated CI/CD and test pipeline that any Copilot agent session can invoke to validate changes to the TellStick add-on (and future HAOS projects) against a real Home Assistant environment without manual intervention.
